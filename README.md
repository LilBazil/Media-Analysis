# Анализ медиа-контента

Веб-приложение на Flask для анализа творческого портфолио или создаваемого контента

Пользователь загружает CSV или XLSX файл с данными о творческих проектах или использует предлагаемые для примера демо-датасеты, задаёт инструкцию или выбирает из шаблонов, после чего LLM-агент проводит анализ и выдаёт текстовый отчёт с анализом и рекомендациями с возможностью вывода графиков

Проект можно использовать для анализа данных, создания плана по постингу контента и аналитики данных о медиа на платформах, где есть желание найти стльные и слабые места и получить рекомендацию по развитию

## Что делает проект

- Загружает CSV/XLSX датасет
- Читает пользовательскую инструкцию
- Подключает LLM через OpenRouter API
- Анализирует полный датасет
- Строит графики через matplotlib
- Показывает итоговый отчёт и журнал действий
- Даёт возможность сохранить результаты в TXT и JSON

## Готовые датасеты

В проекте есть два встроенных примера.


```text
examples/creative_portfolio_demo.csv
```
Датасет содержит 36 строк с данными о творческих проектах. По нему можно проверить, какие категории и стили получают больше вовлечённости, и какие проекты стоит оставить в портфолио


```text
examples/social_content_demo.csv
```
В датасете 60 строк с данными о публикациях на разных платформах, который подходит для анализа контента, форматов, тем, вовлечённости и конверсию

## Входные данные

На вход можно загрузить таблицу в формате CSV или XLSX или выбрать встроенные датасеты, описанные выше

Лучше всего подходят таблицы, где есть:
-название объекта, проекта, публикации или видео
-категория, тема, формат или платформа
-числовые показатели
-дата
-текстовое описание

Пример ввода инструкции:
```text
Проанализируй данные тикток аккаунта.
Какие темы имеют лучшие просмотры, удержание, количесвто лайков и прирост подписчиков.
Дай рекомендации, какие ролики стоит выпускать чаще
```
Также можно выбрать один из предлагаемых шаблонов текстовой инструкции под полем ввода

## Результаты работы

После анализа выводится страниуа, где может быть написано:
-краткий итог с выводами
-рекомендации
-графики
-предпросмотр данных
-полный текстовый отчёт
-журнал действий агента

Результат можно скачать в двух форматах: TXT для чтения и JSON для структурирного результата

## Настройка API

1. Получите API ключ OpenRouter
2. Скопируйте файл `.env.example` в `.env`
3. Вставьте ключ:
```env
OPENROUTER_API_KEY=здесь_писать_ключ
OPENROUTER_MODEL=minimax/minimax-m2.5:free
```
Модель можно заменить на другую доступную в OpenRouter (например, отлично работала gpt5.4 mini, но она будет затратнее)

## Запуск

```bash
python app.py
```
После запуска перейдите в браузер:
```text
http://127.0.0.1:5000
```
